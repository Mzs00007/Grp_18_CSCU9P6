RECORDING TIPS FOR SMOOTHNESS
================================================================================

PRESENTATION STYLE:
  ✓ Speak clearly and at moderate pace
  ✓ Explain what you're doing BEFORE you do it
    ("Now I'll log in as a candidate...")
  ✓ Pause when necessary (let visuals speak)
  ✓ Use cursor to highlight important UI elements
  ✓ Zoom in on small text if needed
  ✓ Show system clock to prove real-time operation

NAVIGATION FLOW:
  ✓ Move deliberately (no frantic clicking)
  ✓ Minimize waiting time (pre-load data if possible)
  ✓ Use multiple user accounts (don't re-login constantly)
  ✓ Have duplicate browser windows open (switch between roles)
  ✓ Keep transitions smooth between use cases

COMMON MISTAKES:
  ✗ Too fast talking (unintelligible)
  ✗ Long silent periods (viewer loses interest)
  ✗ Excessive mouse movement (distracting)
  ✗ Clicking wrong buttons/fixing "mistakes" (looks unprofessional)
  ✗ Showing technical errors without acknowledging them
  ✗ Omitting important validation "Because nobody saw the error message")
  ✗ Rushing through complex workflows (viewers can't follow)

TECHNICAL SETUP:
  • Resolution: Set browser window to standard size (1920x1080 or 1366x768)
  • Font size: Ensure UI text is readable
  • Two monitors? Great - show both (scheduler on one, demo on other)
  • Disable notifications (Slack, email pop-ups)
  • Close browser tabs except those needed
  • Mute volume from other windows
