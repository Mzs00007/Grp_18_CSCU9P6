YOUR SYSTEM'S ARCHITECTURE CONTEXT:
VCFS follows professional MVC (Model-View-Controller) architecture:
  - MODELS (`vcfs.models`): Pure data entities with business logic
  - CONTROLLERS (`vcfs.controllers`): Handle user interactions
  - VIEWS (`vcfs.views`): Java Swing GUI interfaces
  
The 4 test classes focus on the MODELS layer—the foundation that everything
depends on. These 4 classes manage the entire booking workflow:

1. **Reservation.java** - BINDING LINK
   Connects a candidate to an offer at specific time window
   State transitions: CONFIRMED → IN_PROGRESS → CANCELLED
   Critical for preventing double-bookings
   Example: "Zaid booked interview with Google for April 8, 10:00-10:30"

2. **MeetingSession.java** - RUNTIME WRAPPER
   Per-reservation session that exists during the actual meeting
   Manages virtual room, lobby, attendance records
   Lifecycle: Waiting → Active → Ended
   Example: "When Zaid joins at 10:00, system creates MeetingSession linking
            Reservation to VirtualRoom and Lobby"

3. **Lobby.java** - QUEUE MANAGEMENT  
   Waiting area for candidates arriving before their session starts
   FIFO ordering (first in, first out - fair ordering)
   Critical Methods: add(), remove(), listWaiting()
   Example: "If room occupied, candidate waits in lobby until slot available"

4. **VirtualRoom.java** - RESOURCE POOL
   The persistent meeting space (like Zoom room)
   Manages occupancy (who's in the room, capacity limits)
   State: IDLE (empty) ↔ OCCUPIED (meeting active)
   Example: "Google Booth #1 room can host 1 recruiter + 1 candidate at a time"

   @Test
   @DisplayName("add() maintains FIFO order with multiple candidates")
   void testAddMultipleCandidatesFIFO() {
       // Setup
       Candidate cand1 = new Candidate("alice@test.com", "Alice", "pw");
       Candidate cand2 = new Candidate("bob@test.com", "Bob", "pw");
       Candidate cand3 = new Candidate("carol@test.com", "Carol", "pw");
       
       // Execute
       lobby.add(cand1);
       lobby.add(cand2);
       lobby.add(cand3);
       
       // Verify - FIFO order
       Collection<Candidate> queue = lobby.getWaitingQueue();
       Iterator<Candidate> it = queue.iterator();
       assertEquals(cand1, it.next(), "First candidate should be alice");
       assertEquals(cand2, it.next(), "Second candidate should be bob");
       assertEquals(cand3, it.next(), "Third candidate should be carol");
       // Rationale: Ensures queue maintains FIFO ordering, critical for fair
       // candidate management and preventing queue manipulation
   }
