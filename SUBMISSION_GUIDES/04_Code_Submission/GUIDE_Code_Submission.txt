CODE QUALITY EXPECTATIONS
================================================================================

While not graded directly, examiners will review code during demonstration,
checking for:

CODE STYLE:
  ✓ Consistent indentation (4 spaces or tabs, use one consistently)
  ✓ Proper naming (camelCase for methods/variables, PascalCase for classes)
  ✓ Comments for complex logic (especially in model classes)
  ✓ Javadoc for public methods (/** ... */)
  ✓ No dead code or commented-out lines

STRUCTURE:
  ✓ Packages properly organized (vcfs.core, vcfs.models, etc.)
  ✓ Classes have single responsibility
  ✓ Proper encapsulation (private/protected fields)
  ✓ No circular dependencies between classes

FUNCTIONALITY:
  ✓ No compilation errors
  ✓ No obvious runtime crashes (based on demo)
  ✓ Error handling for invalid inputs
  ✓ Thread safety where needed (singletons, shared state)

TESTING:
  ✓ Test classes included
  ✓ Comprehensive coverage (multiple tests per class)
  ✓ Tests are meaningful (not just mocking)

DOCUMENTATION:
  ✓ Class-level Javadoc explaining responsibility
  ✓ Method Javadoc with @param, @return for public methods
  ✓ README if helpful
